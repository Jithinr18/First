Ascend Fit OS

Description

Ascend Fit OS is a next-generation, gamified fitness tracking application designed with a futuristic "Operating System" aesthetic. Unlike traditional fitness apps, Ascend treats the user's health journey as an RPG (Role-Playing Game), allowing users to earn XP, level up, and complete daily quests based on their real-world activity.
Built as a Single-Page Application (SPA) embedded within a portable HTML structure, it features a robust offline-first architecture. The application manages user data (biometrics, logs, preferences) entirely via local storage, ensuring 100% privacy and zero latency. It includes a suite of integrated tools ranging from breathwork visualizers to AI-simulated health coaching.

Features

•	Immersive "System" UI: A sci-fi-inspired interface featuring glassmorphism, neon accents, and fluid animations.
•	Gamification Engine: Earn Experience Points (XP) for every log (Water, Meals, Steps) to level up your profile.
•	Daily Quests: Auto-resetting daily challenges (e.g., "Hydration Hero", "Step Master") to drive consistency.
•	Multi-Language Support: Instant translation support for English, Spanish, French, German, Hindi, and Japanese.
•	Advanced Customization:
  o	Theme Mode: Fully functional Light and Dark modes.
  o	Accent Colors: User-selectable system colors (Neon Green, Blue, Purple, Orange, Red, Pink).
•	Privacy Protocol: A dedicated "Privacy Mode" that blurs sensitive metrics (Weight, BMI) for use in public spaces.
•	Hub Modules:
  o	Mindfulness: Interactive box-breathing visualizer.
  o	Workout Timer: HIIT/Interval timer with Work/Rest phases.
  o	Sleep Lab: Visual analysis of sleep quality trends.
  o	Trophy Room: Unlockable badges for hitting milestones.
•	AI System Coach: An offline, simulated AI chatbot that provides health insights based on your logged data.
•	Data Sovereignty: Full JSON export functionality and a "Reset Protocol" to wipe local data.

How to Run
This application is designed to be lightweight and portable.

Standard Execution

1.	Navigate to the root directory of the repository.
2.	Locate the index.html file.
3.	Open index.html in any modern web browser (Chrome, Firefox, Safari, Edge).
4.	The "System Initialization" (Onboarding) screen will launch automatically.

For Java/Web Server Environments

If deploying within a Java web container (e.g., Spring Boot, Tomcat):
1.	Place the index.html file in your web server's static content directory (e.g., src/main/resources/static or WebContent).
2.	Start your Java server.
3.	Access the application via http://localhost:<port>/index.html.
